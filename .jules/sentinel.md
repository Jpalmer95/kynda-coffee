## 2025-05-29 - [Missing Next.js Middleware Protection for Admin API Routes]
**Vulnerability:** The Next.js middleware in this project protects `/admin` pages, but explicitly ignores `/api/` routes. Several secure `/api/admin/*` endpoints relied solely on this middleware and were entirely lacking manual authentication checks (`getAdminUser`), leaving them vulnerable to unauthorized access and modification (e.g. promo codes, gift card balances, and store settings).
**Learning:** Next.js middleware configurations often purposefully bypass API routes to prevent double-execution or to allow specific API behaviors. It is crucial not to rely exclusively on middleware for API authentication when dealing with sensitive administrative endpoints in an architecture where API routes are excluded from standard path matchers.
**Prevention:** Every secure endpoint under `/api/admin/*` must include a manual authentication check using the dedicated `getAdminUser(req)` helper at the beginning of each handler (GET, POST, PATCH, DELETE) to verify the user is logged in and authorized as an administrator.
